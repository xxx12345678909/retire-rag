## 五、核心模块实现

### 5.1 应用入口（`app.py`）

**关键设计：**

1. **packaging 元数据补丁**（`app.py:31-40`）：Windows 下空目录 `packaging-23.1.dist-info` 遮蔽有效 `packaging-26.2`，导致 `importlib.metadata.version('packaging')` 返回 None，使 `transformers`/`langchain_text_splitters`/`langgraph` 版本检查失败。补丁在所有可能触发 transformers 导入的语句**之前**拦截 `version()`，对 `packaging` 硬编码返回 `26.2`。同一补丁在 `vector_store.py` 与 `query_expander.py` 各重复一次（谁先被导入谁先生效，进程内只 monkey-patch 一次）。

2. **lifespan 预热**（`app.py:73-80`）：FastAPI `lifespan` 启动时实例化 `VectorStoreService` 并调用 `load_document()`，首次请求即可检索。

3. **双启动钩子**：除 `lifespan` 外另有 `@app.on_event("startup")` 初始化认证库、预约库、用户库、健康档案库（`init_db()`）。

4. **CORS**：`allow_origins=["*"]`，原型阶段全开放。

**统一对话入口：**

```python
class ChatRequest(BaseModel):
    query: str
    session_id: str = "default"
    use_agent: bool = False   # True=React Agent, False=ChatService
    profile: str = ""         # 用户档案，不参与意图识别，仅用于 RAG 检索增强

class ChatResponse(BaseModel):
    answer: str
    intent: Optional[str] = None
    kb: Optional[str] = None
    sources: list[dict] = []
    disclaimer: Optional[str] = None
    session_id: str = "default"
```

路由逻辑（`/api/v1/chat`）：
1. `runtime.requests += 1`，生成 `Trace` 记录调用链。
2. `detect_intent(query)` → 命中 `reject_diagnosis` 直接返回拒答文案。
3. `use_agent and react_agent.agent is not None` → 走 Agent；否则走 `chat_service.handle()`。

### 5.2 业务编排（`service/chat_service.py`）

`ChatService` 是业务大脑，单例 `chat_service`。

**`handle(query, session_id, profile)` 主流程：**

```
1. contextual_query = _build_query(session_id, query)   # 拼接历史
2. intent = detect_intent(contextual_query)             # 意图识别
3. rag_query = contextual_query + (" " + profile if profile else "")
4. 按 intent 路由：policy_flow / service_flow / health_flow / general_flow
5. _save_history(session_id, query, answer)
6. result["intent"] = intent; return result
```

**`detect_intent(query)` 意图识别（关键词匹配，方案A）：**

实际优先级（与代码一致）：

```
1. reject_diagnosis  ← 20 个诊断拒答模式（"确诊""开药""处方""吃什么药""怎么治"...）
2. policy            ← POLICY_KEYWORDS（补贴/高龄/津贴/长护险/养老金/社保/户籍...）
3. health            ← HEALTH_KEYWORDS（糖尿病/高血压/用药/护理/康复...）
4. service           ← SERVICE_KEYWORDS（养老院/失能/预约/收费/居家服务...）
5. general           ← 默认，走 platform 库
```

> ⚠️ `chat_service.py:83` 注释写"优先级：health > policy"，但代码实际先判 policy 再判 health（line 97 注释"政策优先于健康，避免档案健康词劫持政策问题"）。**以代码为准：policy 优先于 health。** 此处注释与代码矛盾，属待修小瑕疵。

**四大业务流（统一模式：扩展 → multi_retrieve_context → 拼装 → 免责声明）：**

| 业务流 | 检索 KB | 差异化 | 免责声明 |
|---|---|---|---|
| `policy_flow` | policy | 提取用户画像（`profile_extractor`）+ 条件匹配 | 📌 政策提示 |
| `service_flow` | service | 调 `service_matcher` 做条件筛选推荐 | 无 |
| `health_flow` | health | 强制附加 ⚠️ 医疗免责声明 | 强制 |
| `general_flow` | platform | 空结果引导换方式提问 | 无 |

**多轮对话（`_build_query` / `_save_history`）：**

- 历史存 SQLite（`persist` 模块），非内存。
- 取最近 3 轮拼入查询，助手回答截断 200 字防 token 膨胀。
- `_session_turns` 跟踪每会话轮次。

### 5.3 RAG 检索（`rag/rag_service.py`）

`RagService` 单例 `rag_service`，封装 4 库检索。

| 方法 | 作用 |
|---|---|
| `search(query, kb, k=None)` | 单查询检索；未知 kb 自动回退 platform |
| `multi_search(queries, kb, k=None)` | 多查询检索 + **前 100 字指纹去重**合并 |
| `retrieve_context(query, kb)` | 单查询 → 纯文本上下文（向后兼容） |
| `multi_retrieve_context(queries, kb)` | 多查询 → `(context, docs)` |
| `build_context(docs)` (static) | 文档列表 → `[来源：xxx]\n内容` 拼接；空返回"未找到" |
| `get_sources(docs)` | 去重提取来源标题 + 120 字摘要 |
| `get_kb_stats()` | 各库向量数统计 |

检索器由 `VectorStoreService.get_retriever(kb)` 提供，`search_kwargs={"k": chroma_conf["k"]}`（默认 3）。

### 5.4 查询扩展（`rag/query_expander.py`）

`QueryExpander` 单例 `query_expander`，解决"词汇鸿沟"（口语"高龄补贴" vs 文档"高龄津贴"）。

**`expand(query, n=3)` 流程：**

1. **缓存命中** → 直接返回（LRU dict，上限 200），零 API 调用。
2. **极短词（≤2 字）** → 跳过扩展，返回 `[query]`。
3. **LLM 扩展** → Qwen 生成 n 个规范检索词（口语→正式术语，2-15 汉字，每行一个）。
4. 原始问题始终作为结果第一个元素（保底）。
5. **异常降级** → `_call_llm` 失败时 `return [query]`，不阻塞主流程。
6. 写入缓存 + LRU 淘汰（删 `next(iter)` 即最早插入项）。

`stats` 属性输出命中率（hits/misses/hit_rate），供 `/admin/debug/stats` 展示。

### 5.5 向量存储与去重（`rag/vector_store.py`）

`VectorStoreService` 管理 4 个独立 Chroma Collection（同一 `persist_directory`，不同 `collection_name`）。

**初始化**：为每个 kb 创建 `Chroma(collection_name=..., embedding_function=embed_model, persist_directory=...)`，并构造 `RecursiveCharacterTextSplitter`（参数来自 `chroma_conf`）。

**`load_document(kb=None)` 加载流程（`_load_into_kb`）：**

```
for 每个知识库 (或指定单个):
    data_path = data/{kb}/
    for 每个允许类型文件 (txt/pdf/docx):
        md5 = get_file_md5_hex(path)
        if md5 in md5_{kb}.txt:   跳过（已加载）
        else:
            docs = txt_loader/pdf_loader/docx_loader(path)
            chunks = splitter.split_documents(docs)
            vector_stores[kb].add_documents(chunks)
            追加 md5 到 md5_{kb}.txt
```

**MD5 去重价值**：重启时不重复向量化已处理文档，缩短启动时间。每个库独立 `md5_{kb}.txt`，互不干扰。

**文件加载器**（`utils/file_handler.py`）：`pdf_loader`（PyPDF）/ `txt_loader` / `docx_loader` / `listdir_with_allowed_type` / `get_file_md5_hex`。

### 5.6 模型工厂（`model/factory.py`）

抽象工厂，单例 `chat_model` / `embed_model`：

```python
ChatTongyi(model="qwen-plus", top_p=0.8, temperature=0.1, api_key=os.getenv("DASHSCOPE_API_KEY"))
DashScopeEmbeddings(model="text-embedding-v3", dashscope_api_key=os.getenv("DASHSCOPE_API_KEY"))
```

参数理由：`temperature=0.1` 保证政策事实准确性；`top_p=0.8` 保留候选范围避免机械感；v3 中文语义优于 v2。

### 5.7 ReAct Agent（`agent/react_agent.py` + `agent/tools/`）

**双模式之一**，`use_agent=true` 时启用。

- 使用 LangGraph `create_react_agent`；`__init__` 内 `try/except ImportError` 捕获版本兼容问题，失败置 `self.agent = None`，不阻塞应用。
- 模块级全局 `agent` 同层保护，`app.py` 导入时即降级。
- 6 个工具（`@tool` 注册于 `agent/tools/agent_tools.py`）：4 个 RAG 检索（policy/service/health/platform）+ `get_weather` + `get_user_location`。docstring 即工具说明书，LLM 据此选工具。
- ReAct 循环上限 3 次工具调用。

### 5.8 用户画像提取（`rag/profile_extractor.py`）

`profile_extractor` 单例。从用户问题用 LLM 提取年龄/户籍/社保/特殊群体等结构化条件，供 `policy_flow` 做精确匹配。`app.py` 的 `ChatRequest.profile` 字段允许前端直接传入档案，绕过意图识别、仅增强 RAG 检索。

### 5.9 服务智能导办（`service/service_matcher.py`）

`service_matcher` 单例。`service_flow` 调用，按护理需求/区域/价格等条件从 `institution_db`（6 家示例机构）筛选推荐，区别于纯 RAG 问答。

### 5.10 持久化与会话（`service/persist.py`）

SQLite 持久化：问答日志、会话历史、配置。表结构含 `query/answer/kb/sources/feedback(1/-1)`。`/admin/logs` 与 `/admin/logs/feedback` 直接读写此库。

## 七、数据流（一次 `/api/v1/chat` 请求）

```
1. app.py:chat(req)
   ├─ detect_intent(req.query) → 命中 reject_diagnosis？→ 直接拒答返回
   └─ use_agent? ──► ReactAgent.execute(query)  [分支结束]
        否
2. chat_service.handle(query, session_id, profile)
   ├─ _build_query: 取 SQLite 最近 3 轮历史 + 当前 query → contextual_query
   ├─ detect_intent(contextual_query) → intent
   ├─ rag_query = contextual_query + profile
   └─ {intent}_flow(rag_query):
        ├─ query_expander.expand(rag_query) → [原问题, 扩展1, 扩展2, 扩展3]
        ├─ rag_service.multi_retrieve_context(queries, kb=intent)
        │    ├─ multi_search: 4 路检索 + 前100字指纹去重
        │    └─ build_context: "[来源：xxx]\n内容" 拼接
        ├─ chat_model.invoke(prompt + context) → answer
        └─ 拼接免责声明 / sources
3. _save_history(session_id, query, answer)
4. 写 QA 日志（persist）
5. return ChatResponse(answer, intent, kb, sources, disclaimer, session_id)
```
