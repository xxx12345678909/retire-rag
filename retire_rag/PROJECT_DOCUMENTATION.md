# 🌬️ 空调智能客服系统 - RAG 项目说明文档

## 一、项目基本信息

### 项目名称
**空调智能客服系统** (Air Conditioner Intelligent Customer Service System)

### 应用场景
为空调用户提供实时、准确的产品使用咨询、故障排查、选购建议、维护保养等多方面的智能服务。通过 RAG 技术实现从企业知识库中检索相关内容，结合大语言模型进行专业、流畅的自然语言回答。

### 核心功能
- ✅ **选购指导**：基于用户需求和环境条件推荐合适的空调产品
- ✅ **使用说明**：详细解答空调操作、功能使用等问题
- ✅ **故障排除**：诊断常见故障并提供解决方案
- ✅ **维护保养**：指导用户进行日常维护和清洁
- ✅ **能耗常识**：提供节能建议和能耗相关知识

## 四、数据处理流程

### 4.1 数据加载

**数据来源**：`data/` 文件夹下的知识库文件

```
data/
├── 空调选购指南.txt       (选购相关知识)
├── 空调使用说明.txt       (操作说明)
├── 空调故障排除.txt       (故障解决方案)
├── 空调维护保养.txt       (保养知识)
└── 空调能耗常识.txt       (节能知识)
```

**加载方式**：
```python
from utils.file_handler import txt_loader
docs = txt_loader("data/空调故障排除.txt")
```

### 4.2 数据清洗

**清洗步骤**：

1. **去重处理**
   - 按内容前 100 字符进行去重
   - 移除完全相同的文档

2. **内容验证**
   - 过滤长度 < 10 字符的文档
   - 去除重复字符过多的文本

3. **格式标准化**
   - 去除多余空格和换行符
   - 保留中文标点和英文字符

**关键代码**：
```python
from data_loader_cleaner import DataCleaner

cleaner = DataCleaner()
docs = cleaner.remove_duplicates(docs)
docs = cleaner.filter_documents(docs)
```

### 4.3 文本分块

**分块策略**：使用递归字符分割器 (RecursiveCharacterTextSplitter)

**参数配置**：
```yaml
chunk_size: 100        # 每块约 100 字符（中文约 33 字）
chunk_overlap: 20      # 块间重叠 20 字符，保证上下文连贯
separators:
  - "\n\n"            # 优先按段落分割
  - "\n"              # 其次按换行分割
  - "。"              # 再次按句号分割
  - "，"              # 按顿号分割
  - " "               # 按空格分割
  - ""                # 最后按字符分割
```

**分块示例**：
```
原始文本：
"故障现象：空调不制冷。检测方法：检查温度设置、清洁滤网。
修复方案：调整温度，清洁或更换滤网。"

↓ 分块后 ↓

块1: "故障现象：空调不制冷。检测方法：检查温度设置、"
块2: "查温度设置、清洁滤网。修复方案：调整温度，清洁或"
块3: "或更换滤网。"
```

**处理结果**：
```
5 个源文件 → 500+ 个文本块 → 450+ 个向量
```

## 五、检索方案说明

### 5.1 检索方式

**相似度检索** (Similarity Search)

```python
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
relevant_docs = retriever.invoke("空调不制冷")
```

**工作流程**：
1. 用户问题向量化
2. 计算与库中所有向量的余弦相似度
3. 返回相似度最高的 k 个文档块（k=3）

### 5.2 检索优化

**问题向量化**：
```python
from model.factory import embed_model

query_vector = embed_model.embed_query("空调怎么清洁？")
# 输出：[0.123, -0.456, 0.789, ...] (向量维度: 1024)
```

**相似度计算**：
```
相似度 = (query_vector · doc_vector) / (|query_vector| × |doc_vector|)
       = 余弦相似度 ∈ [-1, 1]
```

### 5.3 检索效果评估

| 指标 | 说明 | 目标值 |
|-----|------|--------|
| **召回率 (Recall)** | 相关文档被检索出的比例 | > 80% |
| **准确率 (Precision)** | 检索结果中相关的比例 | > 75% |
| **MRR (Mean Reciprocal Rank)** | 相关结果排名倒数的平均值 | > 0.8 |

## 六、Prompt 设计

### 6.1 系统提示词 (System Prompt)

**位置**：`prompts/main_prompt.txt`

**核心内容**：
```
你是空调领域的专业智能客服，具备自主的ReAct思考与工具调用能力，
严格遵循「思考→行动→观察→再思考」的流程回答用户问题，
思考过程需贴合空调的应用场景，逻辑自洽。

### 核心思考准则
1. 先判断用户的核心需求，分析当前已有的信息是否足够直接回答
2. 若不足，思考需要调用什么工具获取缺失信息
3. 工具返回结果后，再次判断信息是否完整
4. 若信息足够，整合生成流畅、专业的最终回答
5. 若5次工具调用后仍信息不足，则回复用户：我不知道
```

### 6.2 工具定义

**RAG 检索工具**：
```
工具名：rag_retrieve
描述：从向量库检索空调相关资料
入参：query (检索词)
出参：原始参考资料内容（纯文本，无格式标签）

约束条件：
- 工具仅返回原始资料，不进行总结
- LLM 需要自主思考、整合和重组
- 生成流畅专业的自然语言回答
```

### 6.3 Prompt 示例

**示例1：选购咨询**
```
用户问题：小户型适合哪些空调？

系统思考：
1. 用户需要了解小户型空调选购建议
2. 需要从知识库检索相关信息
3. 使用 RAG 工具查询"小户型空调选购"

系统行动：
调用 rag_retrieve("小户型空调选购")

系统观察：
返回参考资料（选购指南中的小户型相关内容）

最终回答：
基于参考资料，生成包含具体产品、功率、特点的专业回答
```

**示例2：故障排除**
```
用户问题：空调开了但不出风怎么办？

系统思考：
1. 这是空调故障问题
2. 需要获取诊断和解决方案
3. 使用 RAG 工具查询"空调不出风"

系统行动：
调用 rag_retrieve("空调故障 不出风")

系统观察：
返回故障排除方案（检测步骤、修复方法）

最终回答：
整合参考资料，提供清晰的诊断流程和解决步骤
```

### 7.1 技术栈

| 层级 | 技术 | 用途 |
|-----|------|------|
| **应用层** | Streamlit | Web UI |

### 7.2 关键代码结构

```
agent/react_agent.py
├── class ReactAgent
    ├── __init__()              # 初始化智能体
    ├── execute_stream()        # 流式执行查询
    └── 使用 create_react_agent() 创建 ReAct 框架

├── @tool def rag_retrieve()    # RAG 检索工具
├── @tool def get_weather()     # 天气工具（示例）
└── ... (其他工具)

rag/rag_service.py
├── class RagSummarizeService
    ├── retrieve_raw_context()  # 检索原始资料
    └── retriever_docs()        # 获取检索器

model/factory.py
├── ChatModelFactory()          # LLM 工厂
└── EmbeddingsFactory()         # 嵌入模型工厂
```


## 九、系统特色与创新点

### 9.1 架构创新

✨ **LLM 主导设计**
- LLM（大模型）是决策者，RAG 是工具
- 不是 RAG 主导的被动回答
- 真正实现"工具调用"而非"流水线处理"

✨ **纯文本检索结果**
- RAG 返回的是原始资料，不带格式标签
- LLM 进行自主整合和组织
- 避免生成结果"硬生生"地拼接参考资料

### 9.2 性能优化

⚡ **递归分块策略**
- 按段落→句子→空格→字符递归分割
- 保持语义完整性
- 减少语境丢失

⚡ **向量化优化**
- 使用 DashScopeEmbeddings 高质量嵌入
- 维度 1024，精度高
- 支持中英文混合

### 9.3 用户体验

🎯 **流式输出**
- 实时显示生成结果
- 提升用户体验

🎯 **多工具协作**
- RAG 用于知识检索
- 其他工具扩展功能（可选）
